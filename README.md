## Project Organization

```
Вот объяснение структуры вашего проекта на основе предоставленного варианта:

├── LICENSE            <- Открытая лицензия, если выбрана
├── Makefile           <- Makefile с удобными командами, например, `make data` или `make train`
├── README.md          <- README на верхнем уровне для разработчиков, использующих этот проект.
├── data
│   ├── external       <- Данные из сторонних источников.
│   ├── interim        <- Промежуточные данные, которые были преобразованы.
│   ├── processed      <- Финальные, канонические наборы данных для моделирования.
│   └── raw            <- Оригинальная, неизменяемая выгрузка данных.
│
├── docs               <- Стандартный проект mkdocs; см. www.mkdocs.org для деталей
│
├── models             <- Обученные и сериализованные модели, предсказания моделей или резюме моделей
│
├── notebooks          <- Jupyter-ноутбуки. Конвенция именования — номер (для порядка),
│                         инициалы создателя и короткое описание, например,
│                         `1.0-jqp-initial-data-exploration`.
│
├── pyproject.toml     <- Конфигурационный файл проекта с метаданными пакета для 
│                         dismissal_predict и конфигурацией для инструментов, таких как black
│
├── references         <- Словари данных, руководства и все другие пояснительные материалы.
│
├── reports            <- Сгенерированные анализы в формате HTML, PDF, LaTeX и т. д.
│   └── figures        <- Сгенерированные графики и фигуры для использования в отчетах
│
├── requirements.txt   <- Файл требований для воспроизведения среды анализа, например,
│                         сгенерированный с помощью `pip freeze > requirements.txt`
│
├── setup.cfg          <- Конфигурационный файл для flake8
│
└── dismissal_predict   <- Исходный код для использования в этом проекте.
    │
    ├── __init__.py             <- Делает dismissal_predict Python-модулем
    │
    ├── config.py               <- Хранит полезные переменные и конфигурацию
    │
    ├── dataset.py              <- Скрипты для загрузки или генерации данных
    │
    ├── features.py             <- Код для создания признаков для моделирования
    │
    ├── modeling                
    │   ├── __init__.py 
    │   ├── predict.py          <- Код для выполнения предсказаний с обученными моделями          
    │   └── train.py            <- Код для обучения моделей
    │
    └── plots.py                <- Код для создания визуализаций
```
